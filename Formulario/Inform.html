Todos los formularios que estan en internet tienen metodos de programacion por defecto. Estos metodos nos sirven para poder hacer las acciones principales como son: Acciones para poderse conectar con los servidores, a traves de las Arquitecturas Clientes Servidor. Es que vamos a tener por defecto que todos los formularios utilizan el modelo GET, y el segundo metodo principal es POST, aunque existen otros metodos que se encargan del intercambio de informacion. GET: este metodo envia por medio de la URL toda la informacion del formulario traves del navegador. Su principal funcion es obtener los datos del servicio; pero es el metodo mas vulnerable de todos, derivado a que se puede generar inyecciones de codigo. POST: este metodo se encarga de registrar una nueva entrada al sistema, tambien utiliza la URL por envio por defecto, pero su facultad es poder ocultarla, esto significa que los datos no son vicibles para el usuario comun en el navegador.
